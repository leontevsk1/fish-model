# Детальный анализ модификаций и оптимизаций исходного кода (Модель Ва-Тор)

В данном документе подробно описаны все изменения, внесенные в базовую реализацию клеточного автомата Ва-Тор (исходный файл `fishes.c`). Цель модификаций — приведение программы в соответствие с математическими требованиями лабораторной работы, повышение производительности и устранение логических ошибок исходной архитектуры.

## 1. Архитектурные изменения и сбор данных

* **Удаление графической подсистемы (SDL2):** Исходный код был жестко привязан к библиотеке SDL2 для покадровой отрисовки состояний в графическом окне. Весь этот функционал (включая зависимости `sdl_fb.h`, цветовые константы, задержки `sleep_ms`) был полностью вырезан. Это перевело программу из разряда интерактивных графических приложений в разряд высокоскоростных вычислительных симуляций.

* **Внедрение файлового логгирования:** В соответствии с требованиями методических указаний, добавлен вывод данных в формат CSV (`results.csv`). На каждом поколении фиксируется текущее время (шаг), численность рыб и численность акул. Это обеспечивает сырую базу временных рядов, необходимую для построения фазовых портретов и анализа особых точек (Седло, Центр).

## 2. Исправление логических уязвимостей

* **Корректировка вероятностного распределения:** В оригинальном коде логика появления особей при старте содержала грубую ошибку перекрытия интервалов:

  ```

```c
  // ОШИБКА В ОРИГИНАЛЕ
  if(r < 0.1) { /* рыба */ } 
  else if(r < 0.02) { /* акула - никогда не выполнится! */ }
  ```

j
  Вектор ветвления заменен на строгую последовательность непересекающихся порогов (`r < 0.10` для рыб, и `r < 0.12` для акул), что гарантирует корректную стартовую популяцию хищников.

## 3. Десинхронизация жизненных циклов (Сглаживание графиков)

* **Проблема «Ступенчатого» размножения:** В изначальной или наивной реализации все особи появляются с возрастом `bc = 0`. В результате, на шаге `FB` (для рыб) или `SB` (для акул) вся популяция синхронно дает потомство, удваивая численность за один такт. Это создает резкие, неестественные скачки на графиках численности.

* **Решение:** Введена жесткая рандомизация начальных параметров. При старте популяции возраст каждой рыбы устанавливается как `rand() % FB`, акулы — `rand() % SB`, а начальный голод акулы — `rand() % SH`. Это «размазывает» процесс размножения и смертности по времени, приводя графики к плавным колебаниям, характерным для классических дифференциальных уравнений Лотки-Вольтерры.

## 4. Оптимизация производительности (Снижение накладных расходов)

* Изолированная функция `nbr`, вычислявшая доступных соседей, упразднена.

* **Минимизация структуры данных:** Структура `cell` была очищена от неиспользуемых полей. Оставлены только минимально необходимые: тип (`nm`), возраст (`bc`), голод (`hc`) и метка поколения (`flag`).

## 5. Алгоритмическая корректность (Защита от «Бегущей волны»)

* **Однопроходное обновление:** В клеточных автоматах последовательный обход матрицы сверху-вниз может привести к тому, что переместившаяся вниз или вправо особь будет обработана дважды (или более раз) за один такт.

* **Реализация защиты:** Вместо выделения памяти под вторую матрицу (Double Buffering), используется поле `flag`. При обработке клетки `sea[y][x].flag` устанавливается равным номеру текущего поколения `g`. Если алгоритм натыкается на клетку, где `flag == g`, он её пропускает, так как она уже совершила действие на этом шаге. Это позволяет обновлять поле «на месте» (in-place), экономя 50% оперативной памяти и избегая лишних аллокаций.
