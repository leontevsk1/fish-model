### 1. Переработка структуры памяти (Интеграция ресурса)

Вместо создания параллельного массива для хранения еды (что привело бы к промахам кэша и лишним расходам памяти), ресурс интегрирован непосредственно в структуру среды:

```c
typedef struct {
    char nm;    // 'w' - вода, '1' - вид 1, '2' - вид 2
    int bc;     // Счетчик возраста
    int hc;     // Счетчик голода
    int flag;   // Защита от повторного хода
    int food;   // 1 - есть корм, 0 - пусто
} cell;
```

Еда (`food = 1`) теперь является свойством пустой клетки (`nm == 'w'`). Это сохраняет линеаризацию памяти и позволяет оценивать наличие еды и свободного места за один проход по соседям.

### 2. Замена хищничества на конкуренцию

* Логика поиска типа `'f'` (рыба) акулами полностью удалена.
* Введены два новых типа: `'1'` и `'2'`. Оба типа являются «жертвами» собственной физиологии: их счетчик голода `hc` инкрементируется на каждом шаге. Спасение от смерти (`hc >= NH`) зависит только от потребления ресурса `food`.

### 3. Механика возобновляемой кормовой базы

В главный цикл `main` внедрен триггер, срабатывающий через оператор деления по модулю:

```c
if (g % NK == 0) {
    // проход по полю и установка food = 1 для пустой воды
}
```

Это создает импульсный приток ресурса каждые $N_K$ поколений, имитируя рост водорослей. Инкрементируется глобальный счетчик `total_food` для выгрузки в CSV.

### 4. Имплементация асимметричных стратегий поведения

Основная логика в `step` разделена ветвлением `if (type == '1') ... else if (type == '2')`.

* **Вид 1 («Блуждание»):** При нахождении еды происходит стандартный перенос данных ячейки в новые координаты (особь физически шагает в клетку с едой). Старая клетка затирается нулями.
* **Вид 2 («Уничтожение»):** Происходит дистанционное взаимодействие. Если еда найдена в `ny, nx`, алгоритм обнуляет еду в той клетке (`sea[ny][nx].food = 0`), сбрасывает голод особи (`sea[y][x].hc = 0`), но **не выполняет перемещение**. Координаты объекта не меняются, пока вокруг есть еда.

### 5. Обработка коллизии размножения

Стратегия Вида 2 создает конфликт с правилом «потомок остается в старой позиции». Если особь ест не двигаясь, «старой» позиции не существует.
Введен флаг локального состояния `moved`:

```c
if (sea[y][x].bc >= limit_b) {
    if (moved) {
        sea[old_y][old_x] = (cell){type, 0, 0, g, 0}; // Рождение
        sea[y][x].bc = 0;
    }
}
```

Если `moved == 0`, сброс счетчика возраста не происходит, и размножение откладывается до того момента, когда особь будет вынуждена сделать шаг в поисках новой порции еды, тем самым освободив место для потомка. Это устраняет логический баг «перезаписи» соседей при рождении.

### 6. Модификация логгирования

Функция вывода теперь фиксирует 4 параметра: `fprintf(f, "%d,%d,%d,%d\n", g, n1, n2, total_food);`, что позволяет строить графики истощения ресурсов относительно популяционных взрывов конкурирующих видов.
