# Архитектура модифицированной модели Ва-Тор (Трехуровневая система)

Внедрение дополнительного трофического уровня (корма) потребовало пересмотра базовых механик клеточного автомата без усложнения общей структуры данных. Реализована система, в которой рыбы перестают быть обеспеченными бесконечным ресурсом и становятся зависимыми от пространственного распределения водорослей.

## 1. Унификация структуры состояния

Для минимизации потребления памяти и сохранения кэш-локальности структура `cell` не подверглась расширению.

* Счетчик голода `hc`, ранее применявшийся исключительно для контроля популяции акул, теперь универсален. Для типа `'f'` (рыба) он инкрементируется на каждом такте и сбрасывается при поглощении корма.
* Свойство `food` (бинарный флаг) логически привязано к клеткам типа `'w'` (вода). Это исключает необходимость выделения параллельного массива для слоя ресурсов.

## 2. Разделение приоритетов (Двухконтурный поиск)

Алгоритм принятия решений переведен на двухконтурную систему фильтрации доступных узлов. Введены векторы приоритетных (`tr_pri`) и вторичных (`tr_sec`) целей.

* **Для рыб:** Приоритетной целью является вода с флагом `food == 1`. Если таких узлов в окрестности фон Неймана нет, формируется список вторичных целей — пустая вода (`food == 0`).
* **Для акул:** Приоритетной целью остаются рыбы (`nm == 'f'`). Вторичной — любая вода, независимо от наличия в ней корма.
Выбор конкретного узла для перемещения осуществляется через генератор псевдослучайных чисел из сформированного пула наивысшего доступного приоритета.

## 3. Транзитное сохранение среды (Проблема суперпозиции)

Возникла алгоритмическая коллизия: акула, перемещаясь в пустую клетку с кормом, заменяет её своей структурой, а при последующем ходе оставляет за собой «чистую» воду, фактически уничтожая корм, которым не питается.
Для решения внедрен механизм транзитного сохранения:

```c
int keep_food = sea[ny][nx].food;
sea[ny][nx] = sea[y][x];
sea[y][x] = (cell){'w', 0, 0, 0, keep_food};
```

При освобождении узла хищником алгоритм восстанавливает флаг наличия корма, если он присутствовал там до прихода акулы.

## 4. Механика возобновляемого ресурса

Генерация кормовой базы вынесена за пределы функции шага особи (`step`). В главном цикле реализован глобальный триггер роста, срабатывающий с периодом `NK` поколений (операция `g % NK == 0`). При срабатывании триггера происходит линейный проход по матрице с установкой флага `food = 1` для всех узлов типа `'w'`, где ресурс отсутствует.

## 5. Балансировка параметров выживания

Введение смертности рыб от голода потребовало ужесточения начальных условий.

* Лимит голода рыб `FH` установлен больше, чем время до размножения `FB` (20 против 10), чтобы популяция имела математический шанс на воспроизводство при поиске редкого ресурса.
* Начальное заполнение поля кормом повышено до 50% для предотвращения коллапса системы на первых 10 поколениях до первого срабатывания триггера роста.
